PANDUAN UPDATE SISTEM URAT ID V2.0 (AI AUTO-PILOT)
==================================================

1. UPDATE KONFIGURASI
   - File `config.php` telah diperbarui. 
   - JANGAN LUPA memasukkan ulang GEMINI_API_KEY di baris 11 (karena file ini menimpa file lama).

2. FITUR AI BARU (LOGIKA)
   - AI sekarang dikonfigurasi untuk HANYA menerima topik:
     "Youtuber Pemula, Youtuber Sukses, Jasa Seo, Jasa Membuat website, Subscriber Youtube Gratis, Cara Menghasilkan Uang di internet"
   - AI akan membuat artikel panjang (3000-5000 karakter) dengan gaya bahasa marketing.

3. CARA MENGAKTIFKAN POSTING OTOMATIS (CRON JOB)
   Agar AI memposting 1 artikel setiap hari secara otomatis tanpa Anda sentuh:
   
   a. Masuk ke cPanel Hosting Anda.
   b. Cari menu "Cron Jobs".
   c. Tambahkan New Cron Job.
   d. Common Settings: "Once Per Day" (0 0 * * *).
   e. Command:
      /usr/local/bin/php /home/USERNAME_HOSTING/public_html/cron_auto_post.php
      
      (Ganti USERNAME_HOSTING sesuai path hosting Anda. Jika bingung, tanya provider hosting letak path php).

4. TESTING MANUAL
   - Anda bisa mengetes logika AI baru di halaman Admin (admin.php).
   - Klik tombol "Jalankan AI Sekarang (Random Topic)". 
   - Ini akan mensimulasikan proses otomatis yang dilakukan Cron Job.

5. KEAMANAN & INTEGRITAS
   - Sistem deposit, subscribe, dan proteksi (sanksi) tetap berjalan normal.
   - Pastikan cron_protect.php juga tetap berjalan di Cron Job untuk sanksi user yang unsubscribe.

6. PANDUAN MANAJEMEN FOLDER GAMBAR (UPLOADS)
   Sistem menggunakan folder `uploads/` untuk menyimpan bukti transfer deposit. Folder ini krusial agar user bisa melakukan konfirmasi pembayaran.

   A. PEMBUATAN OTOMATIS
      Secara default, script `deposit.php` akan mencoba membuat folder ini secara otomatis dengan izin 0777. Namun, pada beberapa hosting dengan keamanan ketat, ini mungkin gagal.

   B. PEMBUATAN MANUAL (SOLUSI)
      Jika upload gagal, lakukan langkah berikut di File Manager (cPanel) atau SSH:
      1. Buat folder baru bernama `uploads` di root direktori (sejajar dengan index.php).
      2. Klik Kanan folder tersebut -> Change Permissions (CHMOD).
      3. Ubah permission menjadi `755` (User Writable) atau `777` (World Writable - gunakan hanya jika 755 gagal).

   C. HARDENING SECURITY (WAJIB)
      Untuk mencegah hacker mengupload script backdoor/shell seolah-olah gambar, buat file `.htaccess` DI DALAM folder `uploads/` dengan isi:
      
      <FilesMatch "\.(php|phtml|php3|php4|php5|pl|py|jsp|asp|sh|cgi)$">
          Order Allow,Deny
          Deny from all
      </FilesMatch>
      
      Ini akan memblokir eksekusi script berbahaya jika ada yang lolos dari filter PHP.
